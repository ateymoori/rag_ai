Pre-requisites ï Before running any pipeline for deployment against a Windows host, there are a few steps to take for allowing Ansible to connect and make changes to the targeted host. These steps are copied from this Confluence page: Ansible - Windows hosts Windows Server 2008 ï Install .NET Framework 4.0 (or higher) Install Powershell 3.0 (or higher) PowerShell 3.0 is only supported on Windows 7 SP1, Windows Server 2008 SP1, and later releases of Windows. Enable Windows to use WinRM (Windows Remote Management) communication. Ansible provides a powershell script for doing this. https://github.com/ansible/ansible/blob/devel/examples/scripts/ConfigureRemotingForAnsible.ps1 The script will create and use a self-signed certificate for enabling SSL communication via WinRM. Add the domain user SBABs-ansble-prd to the administrators group on the target host. Windows Server 2012 ï Enable Windows to use WinRM (Windows Remote Management) communication. Ansible provides a powershell script for doing this. https://github.com/ansible/ansible/blob/devel/examples/scripts/ConfigureRemotingForAnsible.ps1 The script will create and use a self-signed certificate for enabling SSL communication via WinRM. Add the domain user SBABs-ansble-prd to the administrators group on the target host. Note Ansible might need Python to be installed on the windows machine. Check the documentation of the deployment pipeline, below, if it states that Python must be installed.
